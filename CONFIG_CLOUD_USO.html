// ¿Qué es Thinking y cuándo activarlo? ▼ abrir

Thinking es el razonamiento interno del modelo — lo que Claude procesa antes de responderte. Con Thinking activo, el modelo genera un borrador mental, evalúa ángulos, se auto-corrige y luego responde. Sin Thinking, responde directamente desde sus patrones entrenados.

Sin Thinking (Off)
Respuesta directa desde entrenamiento. Rápido, mínimo consumo. Correcto para tareas predecibles.
Con Thinking (On)
Razonamiento en cadena visible. Más lento, más tokens. Mejor calidad en tareas complejas y ambiguas.
Cuándo activar
Solo con esfuerzo High o Max. En Low/Medium no aporta suficiente para justificar el costo adicional.
Disponible en
Sonnet 4.6 y Opus 4.8 únicamente. Haiku 4.5 no soporta Thinking extendido.
// Reglas de oro · Claude Pro
  • Sonnet + Low es tu default — no lo cambies sin una razón concreta.
  • Opus es una herramienta, no un modelo diario — úsalo como quirófano, no como consultorio.
  • Thinking solo con High o Max — en esfuerzo bajo no justifica el consumo extra.
  • Comprime contexto antes de Opus — usa Archivista para resumir antes de sesiones largas.
  • Max = sesión corta y enfocada — una sesión Max intensiva puede agotar la ventana de 5 horas.
  • Haiku para volumen en API — si integras Make + Claude API a alto volumen, Haiku es el correcto.
